# Contributing to Kawa
 
Thanks for your interest in contributing to Kawa. This guide explains how to get involved before and after the 1 July 2026 launch.
 
## Status
 
Kawa is currently pre-launch. Source code will be published in this repository at the 1 July launch. Until then, the most useful contributions are:
 
- **Joining the waitlist** at [makimoto.ai](https://makimoto.ai) and telling us about your use case
- **Joining our Discord** at [discord.gg/INVITE_CODE](https://discord.gg/INVITE_CODE) to share what you would like Kawa to do well
- **Filing issues** in this repository for documentation gaps, design questions, or capability requests
 
## After 1 July 2026
 
Once the source is published, contributions are welcome through the standard GitHub flow:
 
1. Fork the repository
2. Create a feature branch (`git checkout -b feature/your-feature-name`)
3. Make your changes
4. Run the test suite
5. Commit your changes with a clear message
6. Push to your fork and open a pull request
 
Pull requests are reviewed by maintainers. We will respond within five working days.
 
## Filing issues
 
Use the issue templates in this repository:
 
- **Bug report** for problems with the API, pipeline, or documentation
- **Feature request** for new capabilities or improvements
 
Before filing, please search existing issues to avoid duplicates.
 
## Code of conduct
 
All participants in the Makimoto community, including contributors, maintainers, and users, are expected to follow our [Code of Conduct](CODE_OF_CONDUCT.md).
