# Roadmap
 
This document is a high-level summary of where Kawa is going. The roadmap will be expanded with more specific milestones after the 1 July 2026 launch.
 
## Phase 1: Launch (1 July 2026) ✅
 
- Post-conversation transcription API, hosted in Singapore
- Open-source interactive playground and Python reference client ([`demo/`](demo))
- Five-stage transcription pipeline (audio resampling, voice activity detection with diarisation, noise filtering, speech-to-text, post-processing)
- Orchestration layer published in this repository under the MIT licence
- Developer portal and API documentation at makimoto.ai
 
## Phase 2: Real-time and composability (H2 2026)
 
- Real-time transcription API for live voice agents, captioning, and in-conversation analytics
- Containerised, self-hostable release of the Kawa pipeline for deployment in the customer’s cloud, data-centre, or jurisdiction of choice
- Country-specific managed APIs in additional Asia-Pacific jurisdictions
- First internal pipeline components opened to community contribution
 
## Phase 3: Full transparency (2027)
 
- Progressive opening of all internal pipeline components
- Community-led contribution model for new languages, models, and post-processing logic
 
## Get involved
 
The roadmap is shaped by developer feedback and Discord conversations. If a capability is missing, tell us at hello@makimoto.ai or on Discord.
